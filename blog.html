1. Discuss the scope of var, let, and const


Ans:In JavaScript, var, let, and const are used to declare variables, but they have different scopes and behaviors.


1. var :
  • Variables declared with var have function scope. They are only accessible within the function in which they are defined.
  • They are also hoisted, which means the variable declaration is moved to the top of the function, but the initialization remains in place. 

                                    
  function exampleVar() {
   if (true) {
    var x = 10;
   }
  console.log(x); // 10
                                    
   }
   exampleVar();
  console.log(x); // Error: x is not defined


2. let :
  • Variables declared with const are like let in terms of block scope, but they must be initialized with a value, and that value cannot be changed after assignment. 

                            
 function exampleLet() {
 if (true) {
  let y = 20;
  }
  console.log(y); // Error: y is not defined
  }
  exampleLet();


3. const :
  • Variables declared with var have function scope. They are only accessible within the function in which they are defined.
  • They are also hoisted, which means the variable declaration is moved to the top of the function, but the initialization remains in place. 

                            
  function exampleConst() {
  const z = 30;
  z = 40; // Error: Assignment to constant variable
  }
   exampleConst();


2. Tell us the use cases of null and undefined


Ans: In JavaScript, null and undefined are distinct values with specific use cases:


1. undefined :
  • undefined represents a variable that has been declared but hasn't been assigned a value.
  • It is often the default value for function parameters when no argument is provided.
  • It is also what you get when you access an object property that doesn't exist.
  • JavaScript can also return undefined in various situations where no value is returned. 

                                    
 let x; // x is undefined
 function foo(y) {
 console.log(y); // If no argument is provided, y is undefined
                                    
 }
 const obj = { prop: 123 };
 console.log(obj.nonExistentProp); 
  // Accessing a non-existent property returns undefined

1. null :
  • 'null' is a deliberate value that represents the absence of any object value or the lack of a value.
  • It is often used when you want to indicate that a variable should have no value or when you want to reset a variable to a "nothing" state.
  • It's different from undefined in that it is an explicitly assigned value. 

                                    
 let x = null; // x is intentionally set to null
 const obj = null; // obj is explicitly set to null to indicate no value


3. What do you mean by REST API?

Ans: A REST (Representational State Transfer) API, a widely used architectural style for designing networked applications, allows communication between systems over the internet. It is based on a set of principles that use standard HTTP methods (GET, POST, PUT, DELETE) for data manipulation. REST APIs use resources, represented by URLs, and operate on them using a stateless and uniform interface. They often return data in common formats like JSON or XML. This simplicity and consistency make REST APIs popular for building web services, enabling clients to request and manipulate data from servers, making them a key component of modern web and mobile application development.
